\section{Related Work}
\label{sec:related-work}

\subsection{Agentic AI Frameworks}

Recent frameworks such as LangChain, AutoGen, and CrewAI enable the construction
of multi-agent pipelines where LLMs invoke tools and delegate subtasks to
specialised agents~\cite{langchain2023,autogen2023}. These frameworks provide
routing, memory, and tool-calling primitives but do not enforce methodology
compliance or provide domain-specific governance.

\subsection{Knowledge-Driven Development}

Knowledge-Driven Development (\kdd) is a software engineering methodology that
treats knowledge artefacts --- structured documents, decisions, constraints, and
evidence --- as first-class deliverables~\cite{kdd2024}. Unlike test-driven or
behaviour-driven approaches, \kdd~governs the intent and rationale behind
implementation decisions.

\subsection{Race Engineering Decision Support}

Existing race engineering decision support systems rely on expert systems and
simulation tools~\cite{race-eng-2021}. LLM integration in motorsport has been
explored in preliminary industry pilots but no peer-reviewed architecture has
been published to date.

\subsection{Model Context Protocol}

The Model Context Protocol (\mcp) standardises the interface between LLM agents
and external tools, providing security boundaries, audit logging, and capability
negotiation~\cite{mcp2024}. We use \mcp~as the tool-access layer across all
agent interactions.

\subsection{RAG and CAG}

Retrieval-Augmented Generation (\rag) and Cache-Augmented Generation (\cag)
extend LLM knowledge with retrieved context~\cite{rag2020,cag2024}. We apply
both approaches to inject domain-specific race engineering knowledge into
agent reasoning chains.
